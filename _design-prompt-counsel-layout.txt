# 맥스수시엔진 — counsel 페이지 레이아웃 재설계

## 목표
색/콤보/버튼/토큰은 현재 상태 유지. **배치(레이아웃)만 활용도 높게** 재설계해라. 상담사가 학생 한 명당 평균 5~10분 안에 여러 대학을 비교·저장·PDF 출력하는 업무 UX 를 개선한다.

## 배경
- 체대입시 학원 **수시 상담 페이지**. 상담사 1명 ↔ 학생 여러명 반복 상담.
- 한 학생당 대학 5~10개를 추가해 각 대학별 실기 기록/내신/합산 점수를 실시간 계산하고 저장 + PDF 배포.
- 현재는 topbar 에 학생 선택 + 버튼 몰려있고, 본문은 `<table>` 에 대학별 tbody 가 세로로 쌓이는 구조라 10개 넘어가면 스크롤 지옥.

## 현재 구조 요약
```
topbar  : [page-path] ... [학생콤보 | +대학 | 저장 | PDF]
main
 └ captureArea (PDF 캡처 대상)
    ├ panel-scroll > #collegeTable
    │   └ tbody.row-group (대학 1개 = 3 tr: header/info/practical-row)
    │      헤더:  N. 대학명 [배점표] [간단요강] | 학과명 | 전형명 | 등급 | 내신 | 실기총점 | 합산 | 맥스컷 | 지점컷 | 삭제
    │      info:  [콤보] [콤보] [콤보] [input×6] [삭제]
    │      실기:   종목별 기록/점수 input (1~7개, 가로 flex)
    └ panel > 상담메모 textarea
하단: 저장 버튼 + 성공/실패 msg
모달 2개: scoreTableModal(실기 점수표), collegeDetailsModal(대학 상세 요강)
loadingOverlay (글로벌)
```

## 알려진 문제점
1. **스크롤 지옥** — 대학 8개만 돼도 한 화면 못 봄. 상담사가 "어떤 대학이 제일 유력한지" 전체 조망이 안 됨.
2. **학생 프로필 부재** — topbar 에 이름만. 성별·누적 내신·상담 횟수·마지막 저장 시각 등 맥락 정보 없음.
3. **요약 없음** — 지원 대학 수·군별 분포(수시 전형별)·평균 실기 점수·위험도 등 "이 학생의 지원 그림" 한눈 요약 없음.
4. **실기 행 가독성** — 실기 종목이 테이블 행 내부에 들어가서 종목 폭 부족 + 정보 밀도 과다.
5. **PDF/편집 혼용** — `#captureArea` 가 편집 영역 그대로 PDF 됨 → 편집 UI 를 깔끔히 만들면 PDF 가 이상해지고, PDF 룩을 맞추면 편집이 불편.
6. **연도 맥락 부재** — 사이드바에 연도(26/27) 있지만 main 에서는 "26학년도 상담 중" 같은 signpost 없음.

## 디자인 방향 후보 (이 중 가장 맞는 것, 또는 더 좋은 변형)

### A. 2컬럼 workspace (편집 70% / summary 30%)
- 좌: 대학 편집 테이블 (현재 구조 유지)
- 우: 학생 프로필 카드 + 지원 현황 요약 + 메모 탭
- 상단 학생 콤보는 좌측 상단으로 이동
- 장점: 상담사가 학생 바꿔도 우측 프로필/요약이 자동 갱신

### B. 대학 카드 그리드
- 대학 1개 = 카드 1개 (2~3 col grid)
- 카드 내부: 대학명/학과/전형 콤보(세로 stack) + 실기 + 점수 배지 + "자세히/배점표" 버튼
- 장점: 10개 이상도 한눈 비교
- 단점: 한 행에 다 보이던 컬럼(등급/내신/합산/컷)이 분산됨 → 비교 어려워질 수 있음

### C. 리스트 + 디테일 split
- 좌: 대학 리스트(카드, 요약만 — 이름·합산점수·컷대비)
- 우: 선택된 대학 1개의 디테일 (3단 콤보 + 실기 + 점수)
- 장점: 집중 편집, 비교 쉬움
- 단점: 한번에 여러 대학 보고싶을 때 불편

### D. 하이브리드 (추천 기본안)
- 상단 고정 영역: 학생 프로필 카드(얇게) + 지원 요약 pill (총 N개 · 평균 합산 X · 맥스컷 근접 Y개)
- 본문: 대학 **컴팩트 행** 테이블 (등급/내신/실기/합산/컷을 한 줄에 밀도있게)
- 각 행 클릭 → **하단 drawer** 에 실기 종목 + 배점표 + 간단요강 (지금 모달 2개를 drawer 로 통합)
- 우측 slide panel(토글): 메모 + 히스토리
- PDF 버튼 → PDF 전용 레이아웃 별도 렌더 (편집 UI 와 분리)

## 제약 (반드시 지킬 것)
- 공용 자산 (`assets/css/tokens.css, buttons.css, combobox.css, modal.css, toast.css, sidebar.css, topbar.css, base.css`) 수정 금지.
- `window.api()`, `window.createCombobox()`, `window.showToast()`, `window.openModal()`, `window.escapeHtml()` 등 공용 JS API 시그니처 변경 금지.
- JS 로직 (API 14개, 함수 21개) 변경 최소화 — 데이터 바인딩 selector 만 새 마크업에 맞게 수정.
- **기능 diff 0** — 상담 저장/불러오기/PDF/실기계산/대학 상세 전부 동일하게 동작.
- 하드코딩 컬러 금지 — 전부 `var(--accent)` 등 토큰.
- `counsel.html` (원본) 건드리지 말 것. `counsel.new.*` 만.
- PDF 생성 기능은 보존 필수 (NanumGothic 폰트 유지).
